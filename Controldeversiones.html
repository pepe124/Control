Trabajo práctico: Guía de control de versiones

Seleccionando el programa

Git es un programa plataforma que nos permite compilar un programa y compartirlo con otras personas, creando distintas versiones del mismo programa para mejorarlo. Para utilizar Git en Windows se usa el compilador GitBash.

Instrucciones de git. Limpiar la pantalla: clear

Para subir un archivo: git init (crea un repositorio de git en la
carpeta que esta el archivo a subir).

Para subir un archivo: git add archivo.extencion

Indica que archivo tiene que subir ej txeto.txt es muy importante la extencion.

Para fijarse el estado del git: git status

Para agregar un comentario: git commit -m ”mensaje”

Para agregar el comentario hay que poner comillas y el comentario entre ellas.

Para agregar un archivo de tu computadora para subirlo a git hub se
debe usar el siguiente código: git remote add nombre de la carpeta
en la computadora y el ULR del repositorio de la pagina de git hub al
cual subir, por ej:
git remote add carpeta https://github.com/pepe124/control-de-verisones.git

Para subir el archivo a github: git push –u carpeta master

Lo sube al github, pero antes pedirá el email y la contraseña

Para bajar un archivo: git clone dirección (aqui va la direccion)

Como subir un programa a git hub:

  • Primero abrimos el Git Bash (un programa para trabajar en git)
  • Luego ponemos git clear (para asegurarnos de empezar desde cero)
  • Después ponemos git init para crear un repositorio
  • Se pone qué archivo queremos agregar usando el git add [nombre
    del archivo] con la respectiva extencion
  • Quinto ponemos git push -u nombre de la carpeta master
  • Y finalmente se espera hasta que el programa pida el nombre de
    usuario y la contraseña

Para codificar se necesita un compilador, nosotros usamos Netbeans, que es mucho más fácil de usar que Dev C++ y además permite subir el programa a GitHub sin intermediarios.

Organizando el proyecto

Para organizar el proyecto utilizamos el programa Markdown, que permite organizar mejor el proyecto por temas y es más cómodo de usar que Word. Para realizar la distribución de tareas se usa la tabla Kanban. El Kanban es un modelo de organización con forma de tabla donde todos los integrantes de un grupo participan y colaboran en la producción de un proyecto concreto.

La tabla Kanban se compone de 4 partes:

  • Tarea a hacer (To Do): indica cuál es la tarea pendiente, se marca con un código o número por motivos de prolijidad.

  • Quién lo hace (Doing): Muestra el nombre de la persona a cargo de la tarea, con el número de la tarea al lado.

  • Revisión: Cuando una tarea está completa, primero se revisa antes de marcarla como hecha.

  • Terminado (Finished): Si la tarea se revisó y se tomó como correcta, pasa a esta etapa, aunque puede regresar a revisión si presenta errores con otra tarea.

El Kanban sirve para que todos los integrantes del grupo sepan qué deben hacer y avisarse entre sí qué tarea ya han hecho. Generalmente, hay una persona que, además de trabajar en una tarea, organiza el Kanban. Esta persona se denomina “Scrum Master”; el Scrum Master (SM) debe ver quién sabe hacer mejor determinada tarea y colocarlo en el Kanban. El SM asume el rol de líder, aunque no da órdenes sino que organiza.
